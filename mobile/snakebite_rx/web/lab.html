Backend base URL

Default local API: http://127.0.0.1:8000 (run make api from the repo root). Change below if your backend is on another host or port. Optional query: ?api=http://127.0.0.1:8000. Saved in session storage when you click Apply.

Atomic tests: each wound backbone alone, then fused ensemble, then geo, symptoms, and full multimodal. Snake names use common / generic labels from your geo CSV when present (scientific name shown as secondary). Not clinical ground truth. · Flutter app · API docs

Pipeline — sample image & run everything

GET /test/sample_wound_image returns a JPG from data/geo_data/categories/hemotoxic/. Run all loads that sample (if you have not picked a file), then calls wound → geo → symptoms → full fusion in order.

Wound — fused ensemble

Geo prior only

      
POST /test/geo — Venom-type distribution from GBIF counts for country/state; species ranks from that distribution.

      
      

    


    

      
Symptoms only

      
POST /test/symptoms — Knowledge-base weights; no image.

      
      

    


    

      
Full multimodal fusion

      

        POST /predict — Wound CNN (weighted average of the three softmax vectors) combined with symptoms, geo, and context in log-space.
        Final top class can differ from each backbone when symptoms/geo/context push another venom type.
        Blur / low-detail photos get a retake hint (sharpness score).
